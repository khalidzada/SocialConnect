3. Table Schemas & Relationships

This section describes the table schemas used in Social Connect+ along with their relationships. Tables are designed to ensure data integrity and efficient access.

Users Table
Column Type Constraints

Example: The Users table uses a unique constraint on the Email column to prevent duplicate registrations.

4. Normalization & Constraints

Normalization reduces data redundancy and enhances data integrity. Social Connect+ adheres to the first three normal forms (1NF, 2NF, 3NF).

  • 1NF: Each table has a primary key and atomic values.
  • 2NF: All non-key attributes are fully dependent on the primary key.
  • 3NF: No transitive dependencies exist among non-key attributes.

Example: In the Posts table, the content and timestamp are fully dependent on the PostId, ensuring no redundancy.

5. Indexes & Query Optimization Strategies

Indexes are used to speed up data retrieval. In Social Connect+, indexes are created on frequently queried columns (e.g., Email, CreatedAt).

  • Indexes: Single-column and composite indexes are implemented based on query patterns.
  • Query Optimization: Parameterized queries and caching (using Redis) are used to enhance performance.

Example: An index on the Email column in the Users table drastically reduces lookup time for login operations.
