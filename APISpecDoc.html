1. Introduction

This document defines the communication framework for SocialConnect+. It outlines the available API endpoints, the expected request and response formats (in JSON), the authentication methods employed, and the standardized error handling protocols to ensure secure and consistent interactions between services.

By providing clear and comprehensive API specifications, developers and integrators can build robust and reliable integrations with the platform.

2. API Endpoints
Authentication APIs
  • POST /api/auth/register: Registers a new user.
    Request: JSON payload with username, email, and password.
    Response: JSON status and user ID.
  • POST /api/auth/login: Authenticates a user and returns a JWT token.
    Request: JSON with email and password.
    Response: JSON with a token.
  • GET /api/auth/profile/{id}: Retrieves user profile details.
    Response: JSON with user information.
Social Media APIs
  • POST /api/posts/create: Creates a new post.
    Request: JSON with user ID, content, and optional media URL.
    Response: JSON with post details.
  • GET /api/posts/feed: Retrieves the user feed.
    Response: JSON list of posts.
  • POST /api/posts/{id}/like: Likes a post.
    Response: JSON confirmation.
  • POST /api/posts/{id}/comment: Adds a comment.
    Request: JSON with comment text.
    Response: JSON with comment details.
Messaging APIs
  • POST /api/messages/send: Sends a message.
    Request: JSON with sender ID, receiver ID, and message content.
    Response: JSON with message confirmation.
  • GET /api/messages/conversation/{userId}: Retrieves conversation history.
    Response: JSON list of messages.
E-commerce APIs
  • POST /api/shops/create: Creates a new shop.
    Request: JSON with seller details and shop information.
    Response: JSON with shop details.
  • POST /api/products/add: Adds a new product.
    Request: JSON with product details.
    Response: JSON with product information.
  • POST /api/payments/process: Processes a payment.
    Request: JSON with order details and payment method.
    Response: JSON transaction status.
3. Request & Response Formats

All API communications are performed using JSON format.

Example Request:

{
  \"email\": \"john@example.com\",
  \"password\": \"password123\"
}

Example Response:

{
  \"status\": \"success\",
  \"data\": {
      \"userId\": 123,
      \"username\": \"johndoe\"
  }
}
4. Authentication & Authorization Methods

All secure endpoints require a valid JWT token. Clients must include the token in the Authorization header in the format:

Authorization: Bearer <token>

In addition, OAuth 2.0 is implemented to support third-party authentication providers (e.g., Google, Facebook) for a secure and flexible login process.

5. Error Handling & Response Codes

The API uses standard HTTP status codes:

  • 200 OK: Request succeeded.
  • 201 Created: Resource created successfully.
  • 400 Bad Request: Invalid input or missing parameters.
  • 401 Unauthorized: Authentication failed or missing token.
  • 403 Forbidden: Insufficient permissions.
  • 404 Not Found: Resource does not exist.
  • 500 Internal Server Error: Unexpected server error.

Error Response Example:

{
  \"status\": \"error\",
  \"error\": \"Invalid email or password\",
  \"code\": 401
}
6. Conclusion

This API Specification Document establishes a robust framework for communication between services in SocialConnect+. By detailing all endpoints, request/response formats, authentication methods, and error handling protocols, it ensures consistency, security, and reliability across all integrations.
